Realiza un esquema de transformación para conseguir los siguientes cambios de base: de binario a decimal, octal y hexadecimal, y viceversa

  • Para convertir un número de binario a decimal, hay que asignarle un valor a cada cifra del número binario. De derecha a izquierda, se le asignarán las potencias de 2 —partiendo de 20 de manera ascendente— hasta que no queden más cifras en el número binario. Después, se multiplicará cada cifra binaria con su correspondiente potencia y se sumarán todos los resultados. El número que resulte, será el número decimal de dicho número binario. Por ejemplo, el número 10110011(2 tiene 8 cifras (1 0 1 1 0 0 1 1), por lo que le asignaremos tantas potencias de 2 como cifras tenga, de derecha a izquierda de manera ascendente. Lo colocamos en forma de tabla y procedemos a multiplicar las cifras con su correspondiente potencia:
  • 1 0 1 1 0 0 1 1
    27 26 25 24 23 22 21 20
    128 * 1 = 128 64 * 0 = 0 32 * 1 = 32 16 * 1 = 16 8 * 0 = 0 4 * 0 = 0 2 * 1 = 2 1 * 1 = 1
  • Por último, procedemos a sumar todos los resultados y, si hemos efectuado las operaciones correctamente, resultará el número decimal buscado:
  • 128 + 0 + 32 + 16 + 0 + 0 + 2 + 1 = 179(10
  • Para convertir un número de decimal a binario, aplicaremos el proceso contrario, pero tanteando. Es decir, se buscará en las potencias de 2 qué número es el que más se aproxima al número decimal que queremos transformar. Podrán aparecer dos números, uno mayor y otro menor, pero se escogerá el número menor, ya que las cifras faltantes se irán completando con el resto de potencias. Se realizará la misma secuencia hasta que nuestro número se haya visto reducido a 0. Cuando el número se haya visto reducido a 0, todas las potencias siguientes tendrán un 0 asignado. Por ejemplo, el número 97(10 se aproxima tanto al número 64 como al número 128, pero cogeremos aquel que es menor, el 64. El número 64 sabemos que es 26, por lo que haremos otra tabla con todas las potencias de 26 (la máxima) a 20 (la mínima) en la primera fila y, en la segunda, pondremos 1 o 0 en función de si nos pasamos o no del número a convertir (97):
  • 64 = 26 32 16 8 4 2 1 = 20
    1 1 0 0 0 0 1
  • El número 64 es la potencia de 2 inferior a nuestro número a convertir (97), por lo que se le asignará el primer 1. Hay que proceder a restarle dicha potencia al número decimal para poder continuar con las siguientes cifras (97 – 64 = 33), por lo que ahora necesitaremos una potencia de 2 menor a 64 (26) que no se pase del número resultante, el 33. Como el número 32 puede cubrir dicha posición sin pasarse, le asignaremos un 1, y restaremos nuestro resultado con esta potencia (33 – 32 = 1). El número 16 no lo podemos poner ya que nos pasamos del número 1, así que se le asignará un 0. Esto mismo ocurre con los números 8, 4 y 2, por lo que también asignaremos un 0 a cada uno de ellos. El 1 (20) puede cubrir el 1 resultante, así que pondremos un 1 y efectuamos la resta (1 – 1 = 0). Como ya ha resultado 0, concatenaríamos todas las cifras de 1 y 0 de izquierda a derecha y ya habríamos acabado nuestra conversión. Por tanto, el número resultante es el 1100001(2.
  • Para convertir un número de binario a octal, dividiremos el número binario en cifras de tres, de derecha a izquierda, ya que sabemos que 23 es 8 —el número de valores en base octal, del 0 al 7—, una potencia de la base que estamos tratando. De derecha a izquierda nuevamente, iremos convirtiendo cada grupo de tres cifras a decimal, que será el mismo número en octal, ya que lo máximo que caben en tres cifras binarias es 7 en ambas bases. Una vez convertido cada grupo, concatenaremos todas las cifras resultantes. En caso de que no podamos formar un grupo de tres porque ya no hay más números, se añadirán tantos ceros por la izquierda como sean necesarios para llegar a tres cifras. Por ejemplo, tenemos el número 101110012 que queremos convertir a base octal. Para ello, procedemos a formar grupos de tres cifras: 10 111 001. Como resultan solo dos cifras en la primera división, le añadimos un 0 a la izquierda para formar tres: 010 111 001. Ahora hay que convertir dichos grupos en cifras en base decimal —que serán igual en base octal—:
  • 010 = 2; 111 = 7; 001 = 1
  • Resultan las cifras 2, 7 y 1, por lo que las concatenamos y nuestro número buscado es el 271(8.
  • Para convertir un número de octal a binario, aplicaremos el proceso contrario, es decir, que a cada cifra en base octal le asignaremos su valor en base binaria, asignándole a cada una siempre tres cifras. Por ejemplo, el número 731(8 lo vamos a descomponer de manera binaria, ya que es muy sencillo convertir los números del 0 al 7 a dicha base, asignándole tres cifras binarias a cada cifra octal:
  • 7 = 111; 3 = 011; 1 = 001
  • Concatenamos todas nuestras cifras binarias y resulta nuestro número binario, el 111011001(2.
  • Para convertir un número de binario a hexadecimal, seguiremos la misma mecánica que de binario a octal, solo que en vez de dividir las cifras en grupos de tres de derecha a izquierda, las dividiremos en cuatro, ya que sabemos que 24 es 16 —el número de valores en base hexadecimal, del 0 a la F (15(10)—. De derecha a izquierda nuevamente, iremos convirtiendo cada grupo de cuatro cifras a decimal, teniendo en cuenta que A, B, C, D, E y F en base hexadecimal se corresponden con 10, 11, 12, 13, 14 y 15 en base decimal, respectivamente. Una vez convertido cada grupo, concatenaremos todas las cifras (números y/o letras) resultantes. En caso de que no podamos formar un grupo de cuatro porque no hay más cifras, se añadirán tantos ceros por la izquierda como sean necesarios para que haya cuatro. Por ejemplo, tenemos el número 1010111001(2 que queremos convertir a base hexadecimal. Para ello, procedemos a formar grupos de cuatro cifras: 10 1011 1001. Como resultan solo dos cifras a la izquierda, añadimos dos ceros para formar cuatro: 0010 1011 1001. Ahora hay que convertir dichos grupos en cifras de base hexadecimal:
  • 0010 = 2; 1011 = B; 1001 = 9
  • Resultan las cifras 2, B —recordemos que se corresponde con 11(10— y 9, por lo que las concatenamos y nuestro número buscado es el 2B9(16.
  • Para convertir un número de hexadecimal a binario, aplicaremos el proceso contrario, es decir, a cada cifra en base hexadecimal le asignaremos su valor en base binaria, asignándole a cada una siempre cuatro cifras. Por ejemplo, el número C1A(16 lo vamos a descomponer de manera binaria, asignándole cuatro cifras binarias a cada cifra hexadecimal:
  • C = 1100; 1 = 0001; A = 1010
  • Concatenamos todas nuestras cifras binarias y resulta el número 110000011010(2.

¿Cuántos números distintos se pueden escribir con 8 bits?

Con 8 bits se podrán formar 28 combinaciones posibles, es decir, 256 números distintos —el número de combinaciones del número (0 y 1, que forman 2 combinaciones posibles) elevado al número de casillas o espacios a rellenar (8)—.

El número más pequeño que se puede escribir con 8 bits es 00000000(2, el cual es 0(10.

¿En qué consiste el código ASCII extendido y Unicode?

  • El código ASCII (American Standard Code for Information Interchange, código estándar estadounidense para el intercambio de información) extendido es un estándar de codificación de caracteres que consiste en la representación de texto de 255 caracteres distintos gracias a la combinación de 8 bits (1 byte). Tabla de caracteres ASCII extendido.
  • El código Unicode es un estándar de codificación de caracteres que contiene a su vez los estándares de UTF-8 (8-bit Unicode Transformation Format, formato de transformación Unicode de 8 bits), UTF-16 y UTF-32, entre otros. Estos estándares pueden representar muchos más caracteres que ASCII extendido gracias a que usan un byte para la menor codificación, y cuatro para la mayor —en el caso de UTF-8, para el resto esta cifra va en aumento a medida que hablamos de un UTF mayor—, mientras que ASCII extendido usa solo un byte. En la actualidad, Unicode (versión 12.1) logra soportar más de 137 000 caracteres1, pudiendo en el futuro seguir ampliando este número gracias a sus combinaciones tan bien planificadas. Añadir también que Unicode puede entender ASCII extendido, ya que parte de esos 255 caracteres, con el mismo valor binario. Tabla de caracteres Unicode (UTF-8).
